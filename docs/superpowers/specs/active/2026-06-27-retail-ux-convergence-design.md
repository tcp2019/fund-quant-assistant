> **Status:** active
> **Created:** 2026-06-27

## 目标

在不改动信号引擎与后端 API 的前提下，将产品从「量化研究台」收敛为**普通基民可理解的持仓管家**：用户打开 App 只回答三个问题——**我有多少钱、组合健不健康、这周要不要动、动什么**。

## 背景与问题

v1→v2.5 能力持续叠加，但信息架构仍按功能模块拆分（总览 / 机会 / 信号 / 分析…），导致：

1. **主路径分散**：「该干嘛」在总览、机会、信号三处重复出现
2. **术语偏专业**：再平衡、类内等权、结构待办、相关系数等外露
3. **品牌偏量化**：「基金量化助手」吓退非技术用户
4. **进阶功能平铺**：回测、优化器、宏观、热点与主路径同级展示

引擎逻辑正确；**展示层与导航需要一次产品化收敛**。

## 目标用户（v2.6 主 persona）

| 属性 | 描述 |
|------|------|
| 持仓规模 | 5–30 只公募基金，分散在 1–2 个 App |
| 使用频率 | 每周或每月打开 1–2 次，非日内交易者 |
| 技术能力 | 不会调阈值、不懂夏普/相关系数；能上传截图 |
| 核心诉求 | 「帮我看看要不要调整，调整什么，大概多少钱」 |
| 非目标用户 | 量化研究员、日内交易者、需要自动下单者 |

**部署形态说明（诚实标注）：** v2.6 仍为本机 Web 工具，不解决「普通人不会装 Python」问题；收敛聚焦**已能跑起来的用户**的认知负担。托管部署留 v3+。

## 定位陈述（对外）

### 一句话

> **上传持仓，定期告诉你：组合健不健康、要不要调整、调整什么。**

### 与竞品

| 产品 | 能力 | 我们 |
|------|------|------|
| 养基宝 | 看盘、统计 | ✅ 有；并多一层 **可执行建议** |
| 且慢/各种记账 | 记录 | ✅ OCR 导入类似 |
| 投顾 App | 荐基（持牌） | ❌ 不做；只做 **规则化自检 + 免责声明** |

### 品牌文案（v2.6 起）

| 位置 | 现文案 | 新文案 |
|------|--------|--------|
| 浏览器标题 / Header 主标题 | 基金量化助手 | **基金持仓管家** |
| Header 副标题 | Portfolio overview & signals | **看清持仓 · 知道要不要动** |
| 免责声明（不变） | 非投资建议 | 保持 |

内部仓库名、API 路径、`fund-quant-assistant` 可不变；仅改用户可见文案。

## 信息架构：7 页 → 4 页 + 1 折叠区

### 收敛后导航

```
┌─────────────────────────────────────────────────────────┐
│  总览  │  本周建议  │  持仓  │  设置                      │
└─────────────────────────────────────────────────────────┘
                              │
                    「导入持仓」→ 总览空态 / 设置内入口
                    「深入了解」→ 总览底部折叠 或 /insights
```

| 新路由 | 替代 | 职责 |
|--------|------|------|
| `/` 总览 | 现 Dashboard（精简） | 钱、今日涨跌、结构是否均衡、最多 3 条行动摘要 |
| `/advice` 本周建议 | **合并** `/opportunities` + `/signals` | 唯一「该干嘛」入口：结构问题 → 卖出 → 增配 → 新买 → 全部信号（可展开） |
| `/holdings` 持仓 | 不变 | 明细列表 + 导入入口链接 |
| `/settings` 设置 | 不变（分区简化） | 基础设置 + 「高级策略」折叠 |
| `/insights` 深入了解 | **降级** 现 `/analysis` + 机会页「热点」Tab | 相关性、风险、回测、热点；默认不在主导航 |

### 路由兼容

| 旧路由 | 行为 |
|--------|------|
| `/signals` | 301 式前端 redirect → `/advice?tab=all` |
| `/opportunities` | redirect → `/advice` |
| `/opportunities?tab=themes` | redirect → `/insights?tab=themes` |
| `/analysis` | redirect → `/insights` |
| `/import` | 保留路由，**移出主导航**；总览空态 / 持仓页 / 设置 提供入口 |

## 页面详述

### 1. 总览 `/`（精简）

**保留（首屏）**

- 资产卡片：实时市值、今日盈亏、总盈亏、收益率（现 v2.5 能力）
- 净值异动横幅（有则显示）
- **本周建议摘要**：最多各 1 条（卖 / 增配 / 结构问题），底部链到 `/advice`
- 大类配置饼图（帮助理解「健不健康」）

**移至「本周建议」或折叠**

- HotThemeRadar（热点雷达）→ `/insights?tab=themes`
- ActionSummaryCards 完整三列 → `/advice` 主内容
- ThemeExposurePanel → `/insights` 或总览底部「展开更多」

**保留（次屏，可折叠「持仓与集中度」）**

- 集中度 Top5
- HoldingsTable（或缩为 Top5 + 链到 `/holdings`）

**移除总览重复**

- 不再在总览拉完整 `useOpportunities` + `useHotThemes`（减请求）；摘要接口用 opportunities 限 `limit=1` 或专用轻量字段（**优先复用现有 API 参数**，不新增后端）

### 2. 本周建议 `/advice`（新页，合并机会+信号）

**区块顺序（自上而下）**

1. 免责声明（短版，一行）
2. **结构问题**（原 StructuralAlerts）— 标题改为「先处理这些」
3. **建议卖出**（原 sell_actions + reduce 类 signal 去重展示）
4. **建议增配**（原 buy_actions）
5. **可考虑新买**（原 explore_actions）
6. **全部信号**（折叠区，默认收起）— 原 SignalsTable + 筛选 Tab

**Tab（可选，简化 Opportunities 双 Tab）**

- 默认单页滚动，不设 Tab
- 热点主题整块移至 `/insights`

**空态**

- 无持仓 → 引导导入
- 有持仓未 sync → 「同步数据」按钮（复用 Settings sync）
- 无建议 → 「当前组合无需调整，保持观察即可」

**交互保留**

- AI 解读按钮（SignalsTable 内）
- 候选基金展开（增配信号）
- Sync 后浏览器通知（Settings 开关）

### 3. 持仓 `/holdings`

- 保持 HoldingsTable
- 页头增加「导入 / 更新持仓」按钮 → `/import`
- 可选：显示每只「今日盈亏」列（已有 v2.5）

### 4. 设置 `/settings`

**基础设置（默认展开）**

- 风险偏好模板：保守 / 均衡 / 进取（现 Template）
- 数据同步：手动同步 + 最近同步状态
- 浏览器通知开关
- LLM 解读（v2.6 小步）：API Key + 测试连接（若 v2.6 不含此项则标为 v2.6.1）

**高级策略（`<details>` 折叠，默认关闭）**

- 大类目标权重自定义
- 类内分配模式（等权 / 现占比 / 自定义）
- 规则阈值（再平衡带宽、集中度、业绩阈值等）

折叠标题：「高级策略设置（懂再开）」

### 5. 深入了解 `/insights`（降级原分析页）

**Tab**

| Tab | 内容来源 |
|-----|----------|
| 风险与相关性 | 现 AnalysisPage：相关性热力图、波动率/夏普/回撤 |
| 热点与主题 | 现 Opportunities themes + HotThemeRadar + ThemeExposurePanel |
| 回测与模拟 | BacktestPanel、WhatIfPanel、优化器入口 |

主导航不出现；总览 / 本周建议 底部链接：「想了解为什么？查看深入分析 →」

## 白话文案映射

引擎 `reasons_json` 与 UI 标签统一翻译，**展示层改 copy，不改后端 detail 字符串**（v2.6 优先改 `REASON_RULE_LABELS` 与 StructuralAlerts 标题；detail 原文保留在展开区）。

| 现术语 | 用户看到的说法 |
|--------|----------------|
| 再平衡 / 大类超配 | 「{大类}买太多了」 |
| 大类低配 | 「{大类}买少了」 |
| 集中度 | 「这只基金占组合比例过高」 |
| 高相关 | 「这几只基金走势太像，建议合并」 |
| 结构待办 / consolidate | 「{大类}基金太多（{n} 只），先整理再买入」 |
| rebalance_review | 「距上次调整已超过一年，建议整体看一眼」 |
| 夏普比率 | 「风险收益比偏低」 |
| 最大回撤 | 「近一年跌得比较多」 |
| 超额收益 | 「跑输基准/同类」 |
| 信号 / Signal | **建议**（页面级） |
| 机会 | **本周建议**（页面级） |
| 增配 / reduce | 加仓 / 减仓（或保持增配/减仓，已够白话） |

`signalDisplay.ts` 中 `REASON_RULE_LABELS` 扩展；`StructuralAlerts` 标题「结构待办」→「先处理这些」。

## Onboarding 三步（更新）

| 步 | 现 | 新 |
|----|----|----|
| 1 | 导入持仓 | 不变 |
| 2 | 去设置同步 | 「同步基金数据」（仍链 Settings，或总览 sync 按钮） |
| 3 | 去查看信号 | **「查看本周建议」** → `/advice` |

## 范围

### 包含（v2.6）

1. 导航收敛：4 主导航 + `/insights` 二级
2. 新页面 `AdvicePage`（合并 Opportunities + Signals）
3. 总览精简：摘要化建议、减重复请求
4. 设置分区：基础 / 高级折叠
5. 全局文案：Header、Onboarding、关键组件标题
6. 旧路由 redirect（React Router）
7. README 用户文档同步

### 不包含

- 信号引擎规则变更
- 新 API 端点（复用现有 `/api/opportunities`、`/api/signals`）
- 托管部署 / 安装包
- detail 字段 LLM 批量改写
- 删除回测/宏观/热点能力（仅搬家）

## 验收标准

| # | 项 | 标准 |
|---|-----|------|
| 1 | 主导航 | 仅 4 项：总览、本周建议、持仓、设置 |
| 2 | 主路径 | 新用户 Onboarding 第三步进入 `/advice` |
| 3 | 无重复 | 总览不再展示完整三列 ActionSummaryCards |
| 4 | 兼容 | 访问 `/signals`、`/opportunities`、`/analysis` 自动跳转 |
| 5 | 文案 | Header 为「基金持仓管家」；StructuralAlerts 无「结构待办」字样 |
| 6 | 高级设置 | 类内权重、阈值默认折叠 |
| 7 | 回归 | 后端测试全绿；前端 build 通过 |

## 风险与对策

| 风险 | 对策 |
|------|------|
| 老用户书签失效 | redirect 保留 6 个月 |
| Advice 页过长 | 「全部信号」默认折叠 |
| 总览变「空」 | 摘要卡片 + 大类饼图保留视觉重心 |
